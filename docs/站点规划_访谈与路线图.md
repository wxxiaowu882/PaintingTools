## 1. 文档用途

- **把话说清楚**：我们要做什么、为什么做、先做啥后做啥。  
- **当作“路线图”**：以后讨论和推进都按它来，变更也可以，但要写进记录里，别靠记忆。  
- **当作“协作规则”**：做产品/内容/开发的决定，尽量对照这份文档；如果要走不一样的路，先说明原因，再把结论写回这份文档。

## 2. 运作机制（协作与滚动更新）

这一节只解决一件事：**你干活我帮你推进**，怎么配合最省心、最不容易跑偏。

### 2.1 节奏建议

| 节奏 | 做什么 |
|------|--------|
| **轻量同步** | 你每完成 1～2 个小成果，或者每 **2～3 天**，按模板发一条进度就行（很短也行）。 |
| **周回顾** | 每周一次：看 §3，把完成的勾掉/改状态，把下周要干的写成 3 条以内；然后在 §11 追加一行记录。 |
| **阶段节点** | 比如“第一版发布/课程说明页上线/首发包成型”等，必须在 §11 写一行，也把 §3 的阶段状态改一下。 |

### 2.2 你发进度时建议包含（复制即用）

```
【进度同步】日期：
- 本周期完成：
- 进行中 / 卡点：
- 下一周期打算做（最多 3 条）：
- 是否需要调整路线图优先级（是/否，说明）：
```

不用写长篇；**卡点一句话**就够我帮你拆解、换顺序、排雷。

### 2.3 协作方（AI）在进度同步后做什么

1. **把 §3 更新成“下一步就能做”的样子**：把完成/进行中/待办写清楚，下一动作不超过 3 条。  
2. **把 §11 写一行**：这相当于“项目日志”。大事一定记，小事可以周末一起记。  
3. **必要时才改路线图正文（§6～§9）**：只有当目标/范围/优先级真的变了，才调整正文，并在 §11 写明白“为什么改”。  
4. **给你下一步的执行建议**：顺序怎么排最省力、哪里容易踩坑。若要改代码，按 `.cursorrules` 的“最小改动”来做。

### 2.4 滚动原则

- **先把事实写出来，再改计划**：发现不现实就改 §3（下一步）和 §11（日志），别闷头硬扛。  
- **别靠记忆**：对外怎么说、阶段做到哪，以“本文档 + 仓库现状”为准；聊天里说过但没写进文档的不算定稿。  
- **可持续最重要**：不搞“每天都得交付”的自我消耗；按 §2.1 的节奏来就行。

## 4. 访谈摘要（事实与定位）

### 4.1 对外身份与叙事

- **称呼**：「**童画师吴晓**」。  
- **主打方向**：职业画师，**专画儿童肖像**（水彩肖像是你的核心风格/方向）。  
- **关于“工具是谁做的”**：不用纠结“代码是不是你写的”。你可以直接说：这是你作为画师设计的**学习工具/学习场景**，核心价值在“教学设计”和“审美判断”。  
- **你要被记住的点**：用户记住“童画师吴晓 + 儿童肖像 + 水彩 + 有一套能学会的路径”，比记住“你会写代码”更重要。

### 4.2 个人阶段与动机

- 你现在**没有稳定收入**；以前靠定制画吃饭，但那是“卖时间”，越忙越累，收入也上不去。  
- 你想转到：**知识付费 + 自有平台沉淀 + 让更多人认识你**。  
- 小红书大约 5000 粉，已经有人**反复问你有没有课**：这说明“课程”是有真实需求的。  
- 接下来你会**全力做这个网站**，先把第一版上线，再专心录课。

### 4.3 站点要达成的核心目的（长期）

你做这个站最核心就三件事：  

1. **把想学肖像的人吸引进来**（尤其是对水彩人物/儿童肖像感兴趣的人），而且愿意**反复回来用**。  
2. **帮你的课程“铺路”**：现在先把站和内容体系搭好，等第一版上线后，你会集中精力录课。  
3. **以后做会员（VIP/包月）**：这个计划靠后，但要提前想好“免费/付费的边界”，别到时候大改伤老用户。

### 4.4 内容与生产策略

- **靠什么留住人**：核心是“工具好用”+“场景持续更新”。你会把生产端产出的那些美术知识场景，持续喂给消费端。  
- **AI 怎么用**：AI 可以帮你出“场景草图/灵感/方案”，但**解剖、造型规律这些硬东西必须你终审**（错一处会被放大）。  
- **你已经定了的节奏**：走“可持续迭代”，第一版先跑通闭环，场景数量后面滚动加。

### 4.5 课程形态（意向）

- **时间目标**：你希望**半年内**能把课程挂上去。  
- **录课方式**：可以拆模块录（素描基础、色彩理论、人像解剖、人像光影规律、水彩人物等），再打包成一套卖。  
- **平台与资质的担心**：你担心没有教资会影响在某些平台卖课，这块要你自己去问平台/查规则（本文档不替代合规判断）。  
- **理想状态**：最后是你自己的网站能承接学习（播放/会员），但不急着第一版就把这一整套做完。  
- **第一版要做的**：先上“课程说明页（筹备中）”，把方向说清楚，让想学的人知道怎么关注你。

### 4.6 增长与电商（意向）

- **粘性怎么来**：主要靠“工具本身 + 场景包更新”。场景最好能做成**一条链接就能打开**，方便你发社群/小红书。  
- **淘宝怎么用**：可以卖场景包/素材包当引流品；不建议用“共享账号密码式 VIP”当长期方案（容易传播、难撤销、纠纷多）。  
- **宣传怎么做**：你愿意想文案，但希望 AI 帮你一起写、一起改，还想要“发到哪些平台、怎么发、频率怎么定”的方法。  
- **明确不做**：刷赞刷评这种自动化互动，风险太大（容易限流/封号），不走这条路。

### 4.7 SEO 优先关键词（首版聚焦）

- **先把“童画师吴晓”做稳**：这是最容易拿下、也最重要的。  
- **水彩大词要有心理预期**：水彩人物/水彩教程/水彩人物教程竞争很强。我们更务实的打法是：先做“长尾 + 持续更新 + 站外回链”，慢慢养起来。

## 6. 路线图（主时间轴）

下面是我们要走的“主路线”。可以微调，但改主意要写进 §11（不靠记忆）。

### 阶段 A — 第一版上线（目标：约 15 天窗口，以「可持续迭代」为准）

**目标**：第一版能对外：能用、不卡、能分享，用户一眼知道“这是童画师吴晓的站”，而且后续可以持续加场景。

**第一版建议包含（优先级从高到低）**：

1. **消费端场景 — 可持续迭代下的「首发包」**  
   - 四条线都要有“拿得出手”的代表场景，数量先别硬凑，先保证质量和可用性。  
   - 对应入口（你发链接时会用到）：  
     - **几何体 · 静物**：`Solid.html?sandbox=geometry`（对应 `docs/json/沙盒_几何体静物/`）  
     - **肖像光影沙盒**：`Solid.html?sandbox=portrait`（对应 `docs/json/沙盒_肖像光影沙盒/`）  
     - **头骨 · 骨点 · 肌肉 / 头部造型规律**：`Portrait_c.html`（用 `lib` 等参数区分）  
2. **把“站点是谁”讲清楚**：全局菜单、关键页文案、SEO 基础（标题/描述/品牌词）别跑偏。  
3. **课程说明页（筹备中）**：先把你要教什么、怎么更新、怎么关注你写清楚。  
4. **最小引流闭环**：至少准备 1 条“发出去就能打开”的场景链接，方便你做第一轮引流测试。

**明确不包含（阶段 A）**：完整会员支付、完整站内视频课播放器（可作为阶段 C/D 准备项）。

### 阶段 B — 发布后 2～8 周（滚动迭代）

- 按周/双周节奏加场景、打磨体验，慢慢把数量滚上去。  
- 小红书做固定栏目（比如“本周场景练习”），用 AI 帮你写提纲、改标题、出不同版本，然后回链到站里。  
- 看数据/反馈：哪条线更留人、哪个场景更容易被转发，下周就优先做类似的。

### 阶段 C — 约 6 个月内（课程）

- 第一版站稳之后，把重心转去录课。  
- 同步调研：平台规则/收款路径/合规风险；如果需要，先走“站内展示/试看 + 站外成交”的过渡方案。  
- 站内先把目录、试看、更新机制搭起来，等你准备好再把学习体验逐步收回站内。

### 阶段 D — 远期（VIP / 包月）

- 先把“会员到底给什么”说清楚（权益清单、定价假设、免费/付费边界）。  
- 技术（账号/支付/内容权限）单独立项，别挤压第一版和场景更新节奏。

这张表是你心里的“目标数字”，用来排期和对齐预期。注意：**第一版可以先少一点，但要能演示、能分享，然后按周滚上去。**

| 内容线 | 消费端入口（摘要） | 上线建议（访谈） | 理想规模（访谈） |
|--------|-------------------|------------------|------------------|
| 几何体 · 静物 | Solid · geometry 沙盒 JSON | ≥ **10** 场景再对外宣称「丰富」；首发可少于 10 但需可持续补上 | 越多越好，持续追加 |
| 头骨 · 骨点 · 肌肉 | Portrait_c · anatomy 等 | ≥ **3** 可先行；理想 **10**（可含非纯解剖的头部标注示例） | 持续追加 |
