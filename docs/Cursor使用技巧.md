# Cursor 使用技巧（按重要度排序）

面向日常写代码：先掌握「能明显省时间」的几项，其余按需查阅。

---

## 1. 选对对话方式（最高优先级）

| 方式 | 适合做什么 |
|------|------------|
| **Chat（聊天）** | 问概念、读代码、要小片段说明；少动仓库时用它。 |
| **Composer / Agent** | 跨文件改功能、加文件、跑命令、修构建——**默认优先**，减少「只说不改」。 |
| **Inline Edit（行内编辑，如 Ctrl+K）** | 光标附近几行改名、改格式、小补丁，最快。 |

**习惯**：大改动在 Composer 里一次性说清楚目标、约束、相关路径；避免在聊天里贴整文件让 AI「手抄」。

---

## 2. 用 @ 把上下文喂准（减少胡编）

- **`@文件名` / `@文件夹`**：只把相关代码送进模型，比整仓瞎猜准得多。  
- **`@Codebase` / `@代码库`**：全库语义搜索式提问（「哪里处理了 XXX」）。  
- **`@Web`**：查文档、版本变更、API 最新用法。  
- **`@Docs`**：若已配置官方文档源，可对准某库的说明。

**技巧**：改哪个模块就 @ 那个目录；入口 HTML、插件契约类文件务必 @ 进去。

---

## 3. 项目规则：`.cursorrules` 与 `.cursor/rules`

- **仓库根目录 `.cursorrules`**：本仓库对 AI 的「长期约束」（风格、禁止事项、架构约定）。  
- **`.cursor/rules/`**：可拆成多条规则文件，便于分类维护。

**价值**：写一次，全队/每次对话都生效，减少重复叮嘱和风格跑偏。

---

## 4. Agent Skills（技能包）

在 **Cursor Settings → Rules / Skills** 里可挂载技能（如固定 PR 流程、写规则模板等）。

**价值**：把重复流程固化成「可引用的说明文件」，复杂任务更稳、更省字。

---

## 5. 提问方式：目标 + 约束 + 验收

好例子：

- 「在 `plugins_02/Plugin_XXX.js` 里增加 Y 行为，**不要改** `window` 上已有 API 名；改完说明影响了哪些 HTML。」

差例子：

- 「优化一下」——缺少范围，容易大范围重构。

---

## 6. 索引与忽略：`.cursorignore`

与 `.gitignore` 类似，用于**排除**大资源、生成物、第三方目录，加快索引、减少无关引用。

**适合忽略**：巨型 `node_modules`、构建输出、`.glb` 资产目录（若不需要 AI 读二进制意义）等。

---

## 7. 快捷键（Windows 为主；Mac 多为 Cmd）

具体键位以 **Cursor → Settings → Keyboard Shortcuts** 为准，常见对应：

- **行内编辑**：`Ctrl+K`  
- **打开 Chat**：`Ctrl+L`（或你本地绑定的键）  
- **Composer**：`Ctrl+I`（或 `Ctrl+Shift+I`，视版本而定）

**技巧**：命令面板（`Ctrl+Shift+P`）里搜 `Cursor` 可快速看到所有命令。

---

## 8. 终端与「真环境」

在 Agent 模式下可让 AI **直接跑命令**（安装依赖、跑脚本、看报错）。  

**习惯**：把完整报错日志贴进对话，比「好像报错了」高效得多。

---

## 9. 大改前先提交 Git

Agent 一次改多文件时，**先 commit 或建分支**，方便 `git diff` / 回滚。  

**价值**：心理成本低，敢试 Agent 的大范围修改。

---

## 10. 其它高价值习惯（简短）

- **点名文件路径**：用仓库相对路径，减少 AI 找错文件。  
- **让 AI 先读再改**：「先读 `A` 和 `B`，再改」比直接「实现功能」更稳。  
- **多轮收紧范围**：第一轮探索，第二轮明确「只改某某函数」。  
- **敏感信息**：勿把密钥、令牌贴进聊天；用环境变量或占位符说明。

---

## 推荐阅读顺序

1 → 2 → 5 → 3，然后 6、7、8、9；Skills（4）在规则稳定后再整理即可。
