# Cursor 使用技巧（按重要度排序）

面向日常写代码：先掌握「能明显省时间」的几项，其余按需查阅。

---

## 1. 选对对话方式（最高优先级）

| 方式 | 适合做什么 |
|------|------------|
| **Chat（聊天）** | 问概念、读代码、要小片段说明；少动仓库时用它。 |
| **Composer / Agent** | 跨文件改功能、加文件、跑命令、修构建——**默认优先**，减少「只说不改」。 |
| **Inline Edit（行内编辑，如 Ctrl+K）** | 光标附近几行改名、改格式、小补丁，最快。 |

**习惯**：大改动在 Composer 里一次性说清楚目标、约束、相关路径；避免在聊天里贴整文件让 AI「手抄」。

---

## 2. Settings（窗口右上角齿轮）里的高价值配置

打开 **Cursor → Settings → Cursor Settings**（或齿轮进入的 **Cursor 专属设置**，与 VS Code 的「编辑器设置」不同：后者管主题、字体等，前者管 **AI 行为与模型**）。下面按**对日常收益**排序；侧边栏分类名若随版本微调，以你本机为准。

### 2.1 Models（模型）

| 配置意图 | 说明 |
|----------|------|
| **默认对话 / Agent 用哪个模型** | 复杂重构、多文件用「强」模型；简单问答、补全可用更快/更省配额的模型。 |
| **模型列表与开关** | 关掉不用的提供商或模型，减少选错和干扰。 |
| **API Key / 提供商账号** | 使用自带额度或自备 Key 时在此管理；**勿**把密钥写进仓库或规则文件。 |

**价值**：直接决定答案质量、速度和成本，是第一档设置。

### 2.2 Rules、User Rules（用户规则）

| 配置意图 | 说明 |
|----------|------|
| **User Rules（用户级）** | 对所有项目生效的偏好（例如语言、沟通风格、安全习惯）。 |
| **与 `.cursorrules` 的关系** | 项目规则仍以仓库内文件为准；用户规则作全局补充，避免和项目规则打架时以项目为准。 |

**价值**：不用每次聊天重复「用中文」「不要大改」等个人偏好。

### 2.3 Features（功能总开关）

常见子项（名称可能略有差异）：

| 区域 | 建议关注 |
|------|----------|
| **Chat / Composer / Agent** | 是否自动滚动、是否对「不在上下文里的文件」自动应用修改、迭代中与 Lint/诊断相关的自动化等——按你对「AI 改文件有多激进」调整。 |
| **Cursor Tab（补全）** | 开关与触发习惯；大文件或敏感项目可酌情关闭或适应节奏。 |
| **Editor 相关** | 如行内编辑（Ctrl+K）是否扩大选区、diff 主题等，影响「改代码」手感。 |
| **Terminal** | 终端内 AI 辅助（若提供）的提示与行为。 |

**价值**：把「AI 动手脚」的范围调到你能接受的程度，减少误改、乱跳文件。

### 2.4 Indexing / Codebase（代码库索引）

| 配置意图 | 说明 |
|----------|------|
| **索引开关与范围** | 影响 `@Codebase`、语义搜索速度与准确度；大仓库建议配合 **`.cursorignore`** 排除资源目录。 |
| **忽略路径** | 与项目 `.cursorignore` 一起用，减少噪音和索引时间。 |

**价值**：索引准 → `@` 和「全库问」更靠谱。

### 2.5 Privacy（隐私）

| 配置意图 | 说明 |
|----------|------|
| **隐私模式 / 数据用于改进等选项** | 按公司合规与个人偏好选择；涉密代码先看清条款再决定。 |

**价值**：合规与安心，和「能不能用云模型」直接相关。

### 2.6 MCP（Model Context Protocol）

| 配置意图 | 说明 |
|----------|------|
| **接入外部工具与数据源** | 在设置里添加 MCP Server 后，Agent 可能调用文档站、工单、数据库等（取决于你装的 MCP）。 |

**价值**：团队有统一工具链时收益大；个人可先跳过。

### 2.7 Beta / 实验项

新能力预览；**可能不稳定或改 UI**，适合愿意跟版本的人按需开启。

### 2.8 General（通用）

| 配置意图 | 说明 |
|----------|------|
| **与 VS Code 的关系** | 导入键位/扩展、更新通道等；日常写代码体验多在这里与 **VS Code Settings** 交界。 |

### 2.9 与「齿轮」并列：Keyboard Shortcuts

在 **File → Preferences → Keyboard Shortcuts**（或命令面板搜 `Keyboard`）里改 **Cursor** 相关命令绑定，与 Settings 里功能配合使用。

## 3. 用 @ 把上下文喂准（减少胡编）

- **`@文件名` / `@文件夹`**：只把相关代码送进模型，比整仓瞎猜准得多。  
- **`@Codebase` / `@代码库`**：全库语义搜索式提问（「哪里处理了 XXX」）。  
- **`@Web`**：查文档、版本变更、API 最新用法。  
- **`@Docs`**：若已配置官方文档源，可对准某库的说明。

**技巧**：改哪个模块就 @ 那个目录；入口 HTML、插件契约类文件务必 @ 进去。

---

## 4. 项目规则：`.cursorrules` 与 `.cursor/rules`

- **仓库根目录 `.cursorrules`**：本仓库对 AI 的「长期约束」（风格、禁止事项、架构约定）。  
- **`.cursor/rules/`**：可拆成多条规则文件，便于分类维护。

**价值**：写一次，全队/每次对话都生效，减少重复叮嘱和风格跑偏。

---

## 5. Agent Skills（技能包）

在 **Cursor Settings → Rules / Skills** 里可挂载技能（如固定 PR 流程、写规则模板等）。

**价值**：把重复流程固化成「可引用的说明文件」，复杂任务更稳、更省字。

---

## 6. 提问方式：目标 + 约束 + 验收

好例子：

- 「在 `plugins_02/Plugin_XXX.js` 里增加 Y 行为，**不要改** `window` 上已有 API 名；改完说明影响了哪些 HTML。」

差例子：

- 「优化一下」——缺少范围，容易大范围重构。

---

## 7. 索引与忽略：`.cursorignore`

与 `.gitignore` 类似，用于**排除**大资源、生成物、第三方目录，加快索引、减少无关引用。

**适合忽略**：巨型 `node_modules`、构建输出、`.glb` 资产目录（若不需要 AI 读二进制意义）等。

---

## 8. 快捷键（Windows 为主；Mac 多为 Cmd）

具体键位以 **Cursor → Settings → Keyboard Shortcuts** 为准，常见对应：

- **行内编辑**：`Ctrl+K`  
- **打开 Chat**：`Ctrl+L`（或你本地绑定的键）  
- **Composer**：`Ctrl+I`（或 `Ctrl+Shift+I`，视版本而定）

**技巧**：命令面板（`Ctrl+Shift+P`）里搜 `Cursor` 可快速看到所有命令。

---

## 9. 终端与「真环境」

在 Agent 模式下可让 AI **直接跑命令**（安装依赖、跑脚本、看报错）。  

**习惯**：把完整报错日志贴进对话，比「好像报错了」高效得多。

---

## 10. 大改前先提交 Git

Agent 一次改多文件时，**先 commit 或建分支**，方便 `git diff` / 回滚。  

**价值**：心理成本低，敢试 Agent 的大范围修改。

---

## 11. 其它高价值习惯（简短）

- **点名文件路径**：用仓库相对路径，减少 AI 找错文件。  
- **让 AI 先读再改**：「先读 `A` 和 `B`，再改」比直接「实现功能」更稳。  
- **多轮收紧范围**：第一轮探索，第二轮明确「只改某某函数」。  
- **敏感信息**：勿把密钥、令牌贴进聊天；用环境变量或占位符说明。
