## 场景规划设计

### 日常怎么用这份文档

- **平时干活**：跟着下面的 **「节拍」** 走；只有争议、验收扯不清时，再翻 **「细则」** 找原文。
- **写新场景**：每一环都用同一套步子，避免即兴发挥、临时凑数。

---

## 骨架版：简单、好执行的节拍

### 一页结论（我们到底要堆出什么）

- **一条链**：场景是承前启后的一环；用户要感到在 **升级**，不是在刷碎片。
- **一张卡**：**一个问题 + 一个唯一判断 + 场景里看得见 + 能带到下一关**；说明只负责「指眼睛」。
- **一个爽点**：至少满足 **最小达标**（反直觉、唯一判断、可见证据、视觉爽点）；能加强对比 / 口诀 / 错误预警更好。
- **不凑数**：**数字不先定**；**链条优先**；出现触发信号就做 **复习/综合** 或 **迁移**，不硬堆新环。

### 固定节拍：做一个场景的 5 步（每步都有完成定义）

1. **钉链条**：写清 **前置 / 本环唯一判断 / 证据点（看哪里、看什么变化）/ 后继**；链条说不清 → **不做**。
2. **钉爽点**：设计稿里写清 **主视觉、对比方式、抓眼细节、缩略图策略**（四个问题必答）。
3. **钉文案**：消费端只有 `**标题` + `场景说明`**；说明用 **三行模板**（钩子 → `<b>` 唯一关键 → 一句迁移/收束）。
4. **钉资产**：优先 **同一模型·同一相机·多光位** 复用；**一场景一知识点**，不混讲。
5. **钉发布**：生产端写入 JSON → **批量合并发布**；对照 **最小达标 + 质量底线** 过一遍再对外。

### 换节奏：什么时候不「再加一环」

出现下面 **任一信号**，本节拍改做 **复习/综合环** 或 **迁移环**，而不是继续堆新场景：

- 新环和前面 **太像**，说不清「唯一判断」；
- 文案漂亮但场景里 **证据弱**，看不出来；
- 体验上 **记不住、串不起来**，需要把多个判断打包收束。

## 细则：宗旨与约束（全文）

### 1) 总目标（我们到底在做什么）

- **场景的目的**：把“绘画里最关键、最容易说不清的判断”，做成一条**承前启后**的知识链条。  
  - 每个场景是一环：颗粒度适中、独立成立，但又严格依赖前序知识。  
  - 后面的场景必须以之前的场景为基础，让用户感到“我在升级”，而不是“我在刷碎片”。
- **场景的形态**：每个场景是一张“知识点卡片”，但要做到**完整**：  
  - **一个明确问题**：这一环到底解决哪个困惑？  
  - **一个唯一关键判断**：用户这一关只需要学会“怎么判断”。  
  - **一个可见证据**：在场景里能亲眼看到/验证，不靠空讲。  
  - **一个可迁移结论**：能带到下一个场景、带到更复杂对象（静物/头像）。
- **场景的“闪光点”**（必须追求）：每个场景都要让用户产生至少一种感受：  
  - “恍然大悟 / 原来如此”  
  - “以前怎么没想到”  
  - “真的很有用”  
  - “好精美、好用心”  
  - “快点收藏”
- **通盘范围（统一口径）**：全站菜单里的四条主入口都要设计场景体系，并共享同一套宗旨与风格：  
  - 几何体 · 静物（`solid.html?sandbox=geometry`）  
  - 头骨 · 骨点 · 肌肉（`Portrait_c.html?lib=anatomy`）  
  - 头部造型规律（`Portrait_c.html?lib=form`）  
  - 肖像光影沙盒（`solid.html?sandbox=portrait`）
- **当前推进策略**：先做“几何体主线”，再把方法迁移到另外三条入口。

### 1A) “闪光点”定义（可执行 / 可验收）

> 目标：每个场景都要让人觉得“不一般”，并且愿意收藏反复看。

- **最小达标（必须同时满足）**
  - **一句反直觉**：能打破一个常见误区（让人“哦原来我以前一直错在这”）。
  - **一个唯一判断**：用一句话说清“这一关只训练一个判断：____”（说明里用 `<b>` 钉死）。
  - **一个可见证据**：用户在场景里能“亲眼看到”这个判断成立（不是靠你讲）。
  - **一个视觉爽点**：从感官上先征服用户（“一眼就想点/觉得精致/想收藏”）。
- **加分项（做到 1 条就会很吸引人）**
  - **强对比**：同一物体/同一角度，用一个切换就能看出“对/错差别”。
  - **一句口诀**：短、可记、可复用（用户能脱离场景复述出来）。
  - **错误预警**：指出一个最常见的坑，并告诉用户怎么自检。
  - **镜头爽点**：构图干净、主体明确、层次清晰（缩略图就能看出“质感”）。
- **反例（出现就算失败）**
  - 讲了很多点，但用户说不出“这一关到底练什么”。
  - 说明写得很好，但场景里看不到证据（只能靠想象）。
  - 把“结论”当作“理由”，缺少判断依据（用户学不会迁移）。
  - 视觉上平淡、主次不明、缩略图看不出重点（不想点）。

#### 1A.1 视觉爽点怎么设计（设计时必须写出来）

> 目标：让“闪光点”不是口号，而是在设计阶段就能说清楚怎么做出来。

- **每个场景设计稿必须回答 4 个问题**
  - **主视觉**：用户第一眼看见什么？（主体、角度、画面重心）
  - **对比方式**：爽点来自哪里？（光位对比 / 材质对比 / 边缘对比 / 投影对比 / 标注引导）
  - **抓眼细节**：哪一处会让人“哇/哦”？（高光形状、交界线走向、投影外轮廓、转折边软硬等）
  - **缩略图策略**：缩略图要让人一眼看懂重点（必要时让标注/构图服务缩略图）

### 1B) 数字不先定（内容驱动）

> 目标：不让“环数/复习次数/阶段长度”反过来绑架质量。

- **不预设数字**：不硬性规定“必须多少环”“必须几个复习环”。  
- **数字由内容自然长出来**：当内容需要收束、需要迁移、需要对比时，场景类型自然出现。  
- **宁可少但精**：任何“为了凑数量而做”的场景，直接判定为失败。

### 2) 信息极简（优先级最高）

- **列表展示**：只允许 `标题 + 场景说明` 两块内容。
- **场景说明**：允许换行；只允许用 `<b>...</b>` 做加粗强调；禁止堆结构化字段（如入门/分钟/收益/标签等）。
- **字数与密度**：场景说明默认短句、分行；用 `<b>` 标出唯一关键观察点。
- **完整性不靠堆字**：场景的“完整”主要靠**场景内的可见证据/对比/标注引导**来完成；说明只负责“指眼睛”。

### 3) “可持续产出”约束

- **一场景一知识点**：每个场景只讲一个观察点，不混讲。
- **可复用资产**：优先“同一模型/同一相机，多光位多场景”，降低制作成本。
- **可快速迭代**：内容允许频繁修改；不依赖用户端编辑，统一由生产端维护与发布。

### 4) 学习路径（从基础到进阶）

- **分层设计**：从“看得清 → 画得像 → 画得干净 → 画得对 → 画得高级（材质/光）→ 头像迁移”递进。
- **每层可独立学习**：用户只学某一层也有收获；收藏夹用于常看知识点。
- **链条优先**：没有链条就不新增场景。新增场景必须说明：它承接哪一环、为哪一环打基础。

### 4A) “链条优先”的验收口径（每个场景都要交代清楚）

> 目标：让场景之间真的形成体系，而不是“目录堆叠”。

- **每个场景必须写清四件事**
  - **前置**：这一环依赖哪一环（或哪 2 环）？没有前置就写“无”。
  - **本环唯一判断**：这一关只训练一个判断是什么？
  - **证据点**：用户在场景里应该“看哪里/看什么变化”来验证判断？
  - **后继**：这一环会被哪一环继续用到？（为下一环埋钩子）
- **主链条要求**
  - 每一环都能用一句话说清“我比上一环多学会了什么判断”。
  - 相邻两环之间，尽量只升级一个维度（难度不要跳崖）。

### 4B) 节点类型（让“有机结合”可落地）

> 目标：结构与光影不分家，通过不同“节点类型”自然融合，而不是分成两条线。

- **单一判断环（主力）**：只训练一个判断；必须满足“最小达标三件套”。  
- **复习/综合环（收束）**：不引入新知识，只把前面若干判断串成一个“通关体验”，让体系落袋为安。  
- **迁移环（升级）**：把同一个判断迁移到更复杂对象/更复杂光位，让用户真正学会“举一反三”。

### 4C) 何时插入复习/综合环（内容触发，而不是数字触发）

- 当出现以下信号之一，就优先做“复习/综合环”或“迁移环”，而不是继续堆新场景：  
  - 新场景开始“和前面太像”，很难说清唯一判断。  
  - 说明能写得很漂亮，但场景里证据点不够强（看不出来）。  
  - 用户在体验上“记不住/串不起来”（需要把多个判断打包成一个更大的块）。

### 5) 生产端 → 消费端链路约束

- **唯一内容来源**：内容由生产端写入 JSON（`meta.detail`），消费端只读展示。
- **格式约定**：换行必须保留；`<b>` 仅作为强调标记，其他 HTML 不作为功能。
- **合并发布**：以“批量合并”的统合 JSON 作为发布源，避免手改 JSON。

### 6) 体验约束

- **点击动机**：标题必须“可识别 + 可引发好奇”；说明必须“可执行”（告诉用户要看哪里）。
- **不打扰练习**：进入“写生练习模式”（隐藏标注）时只给一次温柔提示，不打断沉浸。
- **收藏夹定位**：收藏的是“我经常回看的知识点”，不改变原列表排序，仅提供收藏夹视图。

### 7) 命名与文案约束

- **标题规则**：`对象/部位 + 关键观察点`（例：球体：交界线；鼻翼：转折）。
- **说明模板**：  
  - 第 1 行：一句话钩子（“你只需要看…” / “别再…”）  
  - 第 2 行：`<b>唯一关键点</b>`  
  - 第 3 行：一句变化（“看懂它，灰面就不糊。”）

### 8) 质量底线（不做就不发布）

- **知识点明确**：用户看完能复述“我该看什么”。
- **场景可复现**：同一个场景不同设备下观感一致（至少不影响关键观察点）。
- **不以功能堆砌换吸引力**：宁可少场景，但每个场景都“短、准、狠”。
