# 给Gemini的简单说明

## 核心原则
**只优化我明确让你优化的部分，其他所有内容都不许改动。**

## 具体说明

### 1. 什么可以优化？
- **只有**我明确告诉你的优化任务
- 例如：如果我说"优化按钮的颜色"，你就只改按钮颜色
- 如果我说"让分享功能更好看"，你就只改分享相关的UI

### 2. 什么绝对不能动？
- **按钮上的文字**（一个标点符号都不许改）
- **布局结构**（不改HTML标签结构）
- **功能逻辑**（不改JavaScript核心算法）
- **已有的兼容性方案**（不改Canvas尺寸限制等）
- **品牌和文案**（不改"童画师吴晓"等任何文字）

### 3. 工作方式
1. 我会给你明确的任务，比如："优化分享按钮的UI"
2. 你只做这个任务相关的修改
3. 其他所有代码保持原样
4. 修改后确保原有功能100%正常工作

## 举例说明

**正确做法：**
我让你："把分享按钮从黑色改成蓝色"
你只改：`.share-btn { background: blue; }`
其他所有代码都不动

**错误做法：**
我让你："优化分享按钮"
你改了：按钮颜色、文字、位置、功能逻辑
→ 这是不允许的

## 重要提醒
- 宁可少改，不要多改
- 不确定能不能改的，就先问我
- 每次只做我明确要求的优化
- 优化后要测试原有功能是否正常

**一句话总结：我说改哪就改哪，没说的都不要动。**
